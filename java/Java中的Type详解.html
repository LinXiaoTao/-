Java中的Type详解

参考地址

Type相关接口:

  • Type
    Type是所有类型的父接口,子接口有WildcardType,GenericArrayType,TypeVariable<D>和ParameterizedType,实现类有class

  • ParameterizedType
    表示一个参数化类型,比如Collection<String>.有如下方法:

    1. Type getRawType():返回承载该泛型信息的对象
    2. Type[] getActualTypeArguments():返回实际泛型类型列表
    3. Type getOwnerType():返回这个类型的归属者.比如内部类,则返回外部类 
      1. public class TestModel {
      2. public Map.Entry<String,Integer> data;
      3. }
      4. @Test
      5. public void testParameterizedType(){
      6. try {
      7.     Field field = TestModel.class.getDeclaredField("data");
      8.     System.out.println(field);
      9.     System.out.println(field.getGenericType());
      10.     ParameterizedType parameterizedType = (ParameterizedType) field.getGenericType();
      11.     System.out.println(Arrays.toString(parameterizedType.getActualTypeArguments()));//String,Integer
      12.     System.out.println(parameterizedType.getOwnerType());//Map
      13.     System.out.println(parameterizedType.getRawType());//Map$Entry
      14. } catch (NoSuchFieldException e) {
      15.     e.printStackTrace();
      16. }
      18. }

  • TypeVariable
    类型变量,泛型信息在编译时会被转换为一个特定的类型.而TypeVariable就是用来反映在JVM编译该泛型前的信息.有如下方法:

    1. Type[] getBounds():返回类型变量的上边界,若为明确声明上边界,则默认为Object
    2. String getName():返回类型变量名称,即泛型名
    3. D getGenericDeclaration():返回声明该类型变量实体
    4. AnnotatedType[] getAnnotatedBounds()
    1. public class TestModel<T extends TestModel,Y> {
    2.    public T model;
    3.    public Y object;
    4. }
    5. @Test
    6. public void testTypeVariable(){
    7.     try {
    8.         Field model = TestModel.class.getDeclaredField("model");
    9.         Field object = TestModel.class.getDeclaredField("object");
    10.         TypeVariable modelType = (TypeVariable) model.getGenericType();
    11.         TypeVariable objectType = (TypeVariable) object.getGenericType();
    12.         System.out.println(modelType.getName());//T
    13.         System.out.println(objectType.getName());//Y
    14.         System.out.println(Arrays.toString(modelType.getBounds()));//TestModel
    15.         System.out.println(Arrays.toString(objectType.getBounds()));//Object
    16.         System.out.println(modelType.getGenericDeclaration());//TestModel
    17.         System.out.println(objectType.getGenericDeclaration());//TestModel
    18.         System.out.println(Arrays.toString(modelType.getAnnotatedBounds()));//AnnotatedTypeFactory$AnnotatedTypeBaseImpl
    19.             System.out.println(Arrays.toString(objectType.getAnnotatedBounds()));//AnnotatedTypeFactory$AnnotatedTypeBaseImpl
    20.     } catch (NoSuchFieldException e) {
    21.         e.printStackTrace();
    22.     }
    23. }

  • GenericArrayType
    泛型数组,组成数组的元素中有泛型则实现了该接口.它的组成元素为ParameterizedType和TypeVariable类型.它只有一个方法:

    1. Type getGenericComponentType():返回数组的组成对象,即被JVM编译后的实际对象
    1. public class TestModel<T extends TestModel,Y> {
    2. public T[] mArrayT;
    3. public List<String>[] mArraysList;
    4. }
    5. @Test
    6. public void testGenericArrayType(){
    7.     try {
    8.         Field arraysT = TestModel.class.getDeclaredField("mArrayT");
    9.         Field arraysList = TestModel.class.getDeclaredField("mArraysList");
    10.         System.out.println(arraysT.getGenericType());
    11.         System.out.println(arraysList.getGenericType());
    12.         Assert.assertTrue(IsInstanceOf.instanceOf(GenericArrayType.class).matches(arraysT.getGenericType()));
    13.         Assert.assertTrue(IsInstanceOf.instanceOf(GenericArrayType.class).matches(arraysList.getGenericType()));
    14.         GenericArrayType arraysType = (GenericArrayType) arraysT.getGenericType();//T
    15.         GenericArrayType arraysListType = (GenericArrayType) arraysList.getGenericType();//List<String>
    16.         System.out.println(arraysType.getGenericComponentType());
    17.         System.out.println(arraysListType.getGenericComponentType());
    18.     } catch (NoSuchFieldException e) {
    19.         e.printStackTrace();
    20.     }
    22. }

  • WildcardType
    该接口表示通配符泛型.比如? extends Number和? super Integer.它有如下方法:

    1. Type[] getUpperBounds():获取泛型变量的上边界
    2. Type[] getLowerBounds():获取泛型变量的下边界
    1. public class TestModel<T extends TestModel,Y> {
    2. public List<? extends Number> mNumbers;
    3. public List<? super String> mObjects;
    4. }
    5. @Test
    6. public void testWildcardType(){
    7.     try {
    8.         Field numbers = TestModel.class.getDeclaredField("mNumbers");
    9.         Field objects = TestModel.class.getDeclaredField("mObjects");
    10.         ParameterizedType numbersType = (ParameterizedType) numbers.getGenericType();
    11.         ParameterizedType objectsType = (ParameterizedType) objects.getGenericType();
    12.         Type[] numbersArguments = numbersType.getActualTypeArguments();
    13.         Type[] objectsArguments = objectsType.getActualTypeArguments();
    14.         WildcardType numberArgumentsType = (WildcardType) numbersArguments[0];
    15.         WildcardType objectArgumentsType = (WildcardType) objectsArguments[0];
    16.         System.out.println("number");
    17.         System.out.println(Arrays.toString(numberArgumentsType.getLowerBounds()));//
    18.         System.out.println(Arrays.toString(numberArgumentsType.getUpperBounds()));//Number
    19.         System.out.println("object");
    20.         System.out.println(Arrays.toString(objectArgumentsType.getLowerBounds()));//String
    21.         System.out.println(Arrays.toString(objectArgumentsType.getUpperBounds()));//Object
    22.     } catch (NoSuchFieldException e) {
    23.         e.printStackTrace();
    24.     }
    25. }
  • Type及其子接口的来历
    • 泛型出现之前的类型
      没有泛型的时候,只有原始类型.此时,所有的原始类型都通过字节码文件Class类进行抽象.Class类的一个具体对象就代表一个指定的原始类型
    • 泛型出现之后
      泛型出现之后,扩充了数据类型.从只有原始类型扩充了参数化类型,类型变量类型,通配符类型,泛型数组类型.
